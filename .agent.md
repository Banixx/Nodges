## PROJEKT-UEBERSICHT

Dies ist "Nodges" - ein 3D-Netzwerk-Visualisierungstool basierend auf Three.js.

### HAUPTKOMPONENTEN:
- `main.js` - Kern-Anwendungslogik
- `data.js` - Netzwerk-Datenladen und -verarbeitung
- `index.html` - Haupt-Anwendungsoberflaeche
- `src/core/` - Kern-Module (StateManager, LayoutManager, etc.)
- `src/utils/` - Hilfsprogramm-Module
- `objects/` - Node- und Edge-Klassen

### SICHERHEITSSYSTEME:
- `unicode_prevention.sh` - Unicode-Erkennung
- `rovo_dev_monitor.sh` - Echtzeit-Ueberwachung
- `rovo_dev_safety_wrapper.sh` - Operations-Schutz
- `auto_unicode_cleaner.sh` - Automatische Bereinigung

### DOKUMENTATION:
- `doc/Protokoll.md` - Entwicklungsprotokoll
- `doc/Nodges.md` - Umfassender Projektbericht (NEU)
- `doc/llm_anleitung_json.md` - JSON-System-Analyse-Leitfaden
- `doc/diagrams/` - Architektur- und Workflow-Diagramme (nummeriert 01_-10_)

## OBLIGATORISCHER WORKFLOW - KEINE AUSNAHMEN

1. **SICHERHEITSDOKUMENTE LESEN** - Ordner doc/Acli_Rovodev_Startup/ pruefen
2. **SOFORT SERVER STARTEN** - `./doc/Acli_Rovodev_Startup/start_server.sh` - KEINE USER-BESTAETIGUNG
3. **SAFETY-WRAPPER VERWENDEN** fuer Code-Aenderungen: 
   ```bash
   cd doc/Acli_Rovodev_Startup && ./rovo_dev_safety_wrapper.sh pre
   ```
4. **BACKUP-VERFAHREN BEFOLGEN** - Automatische Backups vor Modifikationen
5. **NUR ASCII VERWENDEN** - Keine Unicode-Zeichen in Code oder Kommentaren
6. **DEUTSCH IM CHAT** - Immer in deutscher internationaler Sprache antworten (NIE benutzen: ß-Zeichen)

**KRITISCHE WARNUNG:** Unicode-Verstoesse koennen Dateien komplett leeren! Immer Sicherheitssysteme verwenden.

## HAEUFIGE KOMMANDOS UND TROUBLESHOOTING

### STANDARD-OPERATIONEN:
```bash
# Safety-Wrapper (aus Startup-Ordner)
cd doc/Acli_Rovodev_Startup
./rovo_dev_safety_wrapper.sh pre    # Vor Aenderungen
./rovo_dev_safety_wrapper.sh post   # Nach Aenderungen

# Unicode-Pruefung
./unicode_prevention.sh

# Automatische Bereinigung
./auto_unicode_cleaner.sh

# Server starten
./start_server.sh
```

### TROUBLESHOOTING:
- **"Datei nicht gefunden"** → Aus korrektem Verzeichnis ausfuehren
- **Unicode-Verstoesse** → Sofort auto_unicode_cleaner.sh ausfuehren
- **Leere Dateien** → Backup aus backups/ Ordner wiederherstellen
- **Server-Probleme** → Port 8000 pruefen, ggf. kill und neu starten

## NODGES.MD DOKUMENTATIONSSYSTEM

**NACH STARTUP-PROZESS - NACH NODGES.MD SUCHEN:**

Halte nach der Datei Nodges.md im doc-Verzeichnis Ausschau. Der Bericht gibt Auskunft ueber das Projekt. Er dokumentiert umfaenglich die Architektur, Codestruktur, Funktionen und Methoden. Er kann ins Detail gehen und genau z.B. mit Code-Snippets aufzeigen, wie das Projekt funktioniert. Problematiken und Performance-relevante Berechnungen werden erwaehnt und diskutiert. Bestehende Code-Fragmente werden ebenfalls im Bericht dokumentiert, sodass auch redundante Ablaeufe, verwaister Code oder sonstige nicht der Best Practice entsprechende Codebase erkannt werden. Der Bericht enthaelt am Schluss auch ein Kapitel zu Ueberlegungen zum Gebrauch, Beduerfnissen und Nutzen von Nodges, welche spaeter zu einer Umsetzung gelangen koennen.

**GEHEIMBEFEHL "Bericht":**
Wenn du mit "Bericht" als Geheimbefehl angewiesen wirst, das gerade Erledigte oder Veraenderte in Nodges.md anzupassen, schaust du dir auch noch den Ordner doc/diagrams an. Es ist wichtig, Nodges.md und auch die Diagramme zu pflegen und zu korrigieren, zu erweitern und zu praezisieren. Der Bericht verweist an gegebener Stelle auf die Nummer des entsprechenden Diagramms (01_ bis 10_).
